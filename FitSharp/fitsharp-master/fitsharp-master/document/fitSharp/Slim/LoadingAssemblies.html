Loading Assemblies
To execute our tests, Slim needs to find the assemblies that contain your code. There's two ways to do this.
  • We can list the assemblies in PATH definitions.
!path c:\myfolder\mytest.dll
!path c:\myfolder\myother.dll
  • We can list the assemblies in a Suite Configuration File. We can also specify namespaces here, or in a Slim Import table.
<?xml version="1.0" encoding="utf-8" ?>
<suiteConfig>
  <ApplicationUnderTest>
    <addAssembly>c:\myfolder\mytest.dll</addAssembly>
    <addAssembly>c:\myfolder\myother.dll</addAssembly>
    <addNamespace>My.Namespace</addNamespace>
  </ApplicationUnderTest>
</suiteConfig>
This file is then specified in the COMMAND_PATTERN definition.
!define COMMAND_PATTERN {%m -c c:\myfolder\mysuite.config.xml -r fitSharp.Slim.Service.Runner %p}
